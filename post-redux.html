Simplest guide to React + Redux - part 0

(! in this article is going to be used React and later it's combination with Redux, so I guess, you should have some knowledge abouts React's principles)

The way I learn things is never simple, because I am too lazy to start with simple examples, I always need to start with the complex ones and then start to cry. And it was same for Redux. 

At the beginning I started to write simple user interface in React, I guess it is important to note, that back then I even didn't know much about Javascript. But I was like, hey let's create something in this new and popular Javascript framework. So I created some extremely simple components, but then I needed a form and that was the time, when things started to go sideways. 

I would guess that in any "normal" React tutorial, when it's time to talk about forms, they will actually create the form. For example like this(let's say for simplicity, that you use Bootstrap components):

            
<Form>
  <Label>{props.formFieldLabel}</Label>
  <Input
    name="formfield"
    id="formfield"
    placeholder=""
    value={props.value}
    onChange={props.onChangeNewFormField} />
</Form>

Here it is a simple example of a React form with data and actions stored in props, which are passed from the higher component, a container.

But in that time I didn't know this and like I said before, I was just lazy. So I told myself: "there has to be some package to solve this problem". And actually there is and it's called: "Redux Form". 

(And that was the point, when everything started to go sideways)

 

 Firstly, when you look at the first example, called: "Simple Form" ( :-) ), it looks not so bad. There are some fields of the form, some props flags and handle submit and reset functions (when I saw this for the first, I didn't even know how it works). But you can at least say to yourself: "ok, this is the form it has some functions and flags, where are they defined, we don't know though".

When you accept, that there is something called: "props" and it's comming from somewhere (I guess this is problem with some tutorials, when they show you only part of the example, probably it's most important, but still it's just a part). But at the end, there is this:

          
export default reduxForm({
  form: 'simple' // a unique identifier for this form
})(SimpleForm)

 In the documentation to this function you can read: 

"Creates a decorator with which you use redux-form to connect your form component to Redux. It takes a config parameter which lets you configure your form."

Which can be with certain dose of imagination rewritten in something like this (this is huge simplification, take this lightly).

            
reduxForm(connect({
  // redux-form configuration
})(SimpleForm));

With this as the most important part: 

connect()(SimpleForm)
.

From my short experience I have to say, that when you work with Redux, this part could be the most important one. 

What it is? How does it work? Why it has such hideous notation? Simply put, this is the way, which secure that in this case component SimpleForm will be "connected" to Redux mechanism.
